Diseño de Base de Datos

Estructura normalizada hasta la tercera forma normal para eliminar redundancias y garantizar integridad de datos.

Tabla Usuario
IdUsuario
INT, PRIMARY KEY, IDENTITY
NombreUsuario
VARCHAR(50), NOT NULL, UNIQUE
Contraseña
VARCHAR(100), NOT NULL
Rol
VARCHAR(30), NOT NULL
Tabla Alumno
IdAlumno
INT, PRIMARY KEY, IDENTITY
Nombres
VARCHAR(80), NOT NULL
Apellidos
VARCHAR(80), NOT NULL
Edad
INT, CHECK (Edad > 0)
Carrera
VARCHAR(60)

Procedimientos Almacenados

Implementación de 5 procedimientos almacenados por cada tabla para operaciones CRUD completas:

  • Listar registros
  • Agregar nuevos registros
  • Actualizar registros existentes
  • Eliminar registros
  • Buscar registros específicos

Características implementadas:

  • Validaciones internas para prevenir duplicados
  • Manejo de errores con TRY...CATCH
  • Transacciones para operaciones críticas
  • Parámetros tipados para evitar SQL Injection

Interfaz de Usuario

Desarrollo de formularios de escritorio en C# con Windows Forms para gestionar las entidades:

  • Formulario de Usuario (frmUsuario)
  • Formulario de Alumno (frmAlumno)

Funcionalidades implementadas:

  • Registro de nuevos elementos
  • Modificación de datos existentes
  • Eliminación con confirmación
  • Búsqueda y filtrado
  • Visualización en DataGridView

Seguridad y Validaciones

Implementación de validaciones en múltiples capas para garantizar la integridad de los datos:

Validaciones en Cliente

  • Campos obligatorios
  • Tipos de datos correctos
  • Formatos específicos
  • Confirmaciones antes de eliminar

Validaciones en Servidor

  • Existencia de registros
  • Prevención de duplicados
  • Transacciones atómicas
  • Integridad referencial
